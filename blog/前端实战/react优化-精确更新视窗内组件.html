# react优化-精确更新视窗内组件

在做后台管理系统的时候，通常第一页dashbord都是一些图表类的统计数据。随着业务的更迭和老板的逼迫，会发现页面需要展示的图表统计类组件越来越多，通常一屏是展示不完的。在某个组件数据需要更新的时候，愉快的点下更新按钮发请求拿到新的数据，数据更新然后界面刷新。然而，通常情况下，是有多个组件的数据依赖同一条接口的。所以实际上更新组件A的时候，也许组件B、C、D、E都更新了，但是B、C、D、E却还需要鼠标滚轮滚个几转才能看到。可以说，组件B、C、D、E的更新是无意义的，如果其中有着较复杂耗费时间的逻辑的话，还会影响性能。

所以，目标就很明确了，我想要做的事情不仅是当出现在视窗内的时候才去加载组件，我还想做到，当外部数据改变的时候（也就是通过props传进去的数据）在视窗内才去更新组件。

是的，shouldComponentUpdate 可以让开发者去控制组件的更新。可是很多时候组件所依赖的数据并不是一个基本类型的值，{}==={} 为 false，是的，面对这种情况，优秀的开发者们也是一定能想到办法，对比两个不同的对象时去做特殊处理。但其实，我们不用去关心具体依赖的对象到底做了什么变化，我们只关心当组件依赖的外部数据改变的时候，它是否处于可见窗口中。如果当前外部数据更新了的时候，组件不处于可见窗口中，组件就不刷新，等到组件进入可见窗口，在执行更新的操作，也许这时外部数据已经刷新了4次、5次或者只有一次，这些细节都并不需要去关心。

本文将涉及到将控制更新的逻辑抽离为HOC高阶组件（知道现在hooks很火，但是恕我无能，这个场景我实在不知如何用hooks解决）、react一些生命周期、api的运用、以及在使用 typescript 的情况下，如何纠正在使用了高阶组件后props参数泛型。

# 2、获取真实的dom节点, findDOMNode。

能调用 getBoundingClientRect 这个api的一定是一个真实的dom节点。我首先想到了ref， 但是马上又意识到不行，因为ref只有在 HostComponent 类型的组件也就是类似 <div></div> 这样的组件上使用才是真实的dom节点，如果是 classCompoennt 类型的组件，ref就是这个class的实例。

翻看 react 的官方文档，找到了 findDOMNode 这个api，

import { findDOMNode } from 'react-dom'

这个api的作用就是找到当前这个节点下的第一个dom节点。当前，如果当前的节点已经是一个dom节点了就不需要在往下找了。用几张图来简单讲讲这个api： 比如这张图片中的结构，react的虚拟dom树父组件的child属性指向第一个子节点，子节点的 return 属性指向父节点，子节点的 sibling 属性指向兄弟节点，堆Parent节点使用 findDOMNode api的话， 最终找到的真实的dom节点是图中蓝色的节点。 因为这个查找的过程是深度优先遍历的， react先沿着child属性一直往下找，一直找到 child 属性为null了，就从最后遍历的到的节点的兄弟节点开始找，然后又重复这个过程。

# 3、编写HOC

为什么要写一个HOC呢？直接把控制更新到逻辑写到要使用到组件里不可以吗？ 这样做是可以到，但是，如果后续其他组件想要使用也得去重复粘贴一遍代码，写成一个HOC后后面如果在需要使用到这个能里，只需要用HOC包一下就行了。 其次，我认为，控制更新的这一块逻辑，本身就是不属于原本的组件的，而且HOC真的很适用于这个场景，因为我们都知道react的更新是至上而下的，子组件有更新必定是父组件有更新，可以很方便的使用 shouldComponentUpdate 去控制要不要更新子组件，子组件自己内部的更新也可以与这一块儿逻辑隔离开来。

直接上代码，讲解放到代码中更清晰：

import React, { ReactFragment, Ref, ReactElement } from 'react'
import { findDOMNode } from 'react-dom'

interface LazyLoadComState {
  shouldLoad: boolean;
}

type State = Readonly<LazyLoadComState>

function throttle(func: Function, timeout: number):Function{
  let timer: any
  let prevHandTime: number = 0
  return function(...args){
    let now = Date.now()
    if (timer) {
      clearTimeout(timer)
    }
    if ( now - prevHandTime >= timeout ) {
      func.apply(this, args)
      prevHandTime = now
    } else {
      timer = setTimeout(()=>{
        func.apply(this, args)
        prevHandTime = now
      }, timeout - (now-prevHandTime))
    }
  }
}

export default <P extends object>(
  WrapComponent: React.ComponentType<P>
  ): React.ComponentType<P> =>{
  return class LazyLoadCom extends React.Component<P, State> {
    static displayName = WrapComponent.displayName || WrapComponent.name || 'LazyLoadCom';
    dom: ReactElement | HTMLDivElement // 子组件ref
    instanceDom: Element // 真实的dom节点
    hasPropsChange: boolean // props是否发生改变
    listenerHander: any   // 监听窗口改动
    constructor(props){
      super(props)
      this.state = {
        shouldLoad: false // 开始默认不加载子组件
      }
      this.listenerHander = throttle(this.listener, 300).bind(this)
    }

    componentDidMount(){
      this.instanceDom = findDOMNode(this.dom)
      // 组件加载的时候检查一次，如果组件在视窗内 要将组件加载
      if ( this.checkDomInView() ) {
        this.setState({
          shouldLoad: true
        })
      }
      window.addEventListener('scroll', this.listenerHander)
    }

    componentWillUnmount(){
      window.removeEventListener('scroll', this.listenerHander)
    }

    listener = ()=>{
      if ( this.checkDomInView() ) {
        // this.state.shouldLoad==false 的情况是组件还没有加载过
        if ( !this.state.shouldLoad ) {
          // 组件加载
          this.setState({
            shouldLoad: true
          })
        } else {
          // 说明组件已经加载过了，这次出现在视窗也判断到 props发生改变了，所以强制更新
          // 组件更新
          if ( this.hasPropsChange ) {
            this.forceUpdate()
          }
        }
      }
    }
  
    componentDidUpdate(){
      // 这个生命周期在HOC更新后调用，既然是已经更新了 那子组件强制更新的指令也下达了，所以这里将
      // hasPropsChange 改回 false
      if ( this.hasPropsChange ) {
        this.hasPropsChange = false
      }
    }
    shouldComponentUpdate(nextProps){
      let visible = this.checkDomInView()
      if (!(nextProps==this.props)) {
        // 很关键的一点，判断组件依赖的数据是否改变。
        this.hasPropsChange = true
      }
      // 如果组件没有出现在视窗里就返回false，略过这一次更新。
      // 这里就是为了阻断 外部props改变导致的更新
      return (visible)
    }

    checkDomInView(){
      let rect = findDOMNode(this.dom).getBoundingClientRect()
      const yInView = rect.top < window.innerHeight && rect.bottom > 0
      const xInView = rect.left < window.innerWidth && rect.right > 0
      return yInView && xInView
    }
    render(){
      if ( this.state.shouldLoad ) {
        return (
          <WrapComponent ref={el=>this.dom=el}  {...this.props} />
        )
      } else {
        // this.state.shouldLoad == false 的情况使用一个 占位替代一下
        // 其实可以考虑将这一块也做成 参数提出来外部传入， 不过我懒得写了。
        return <div ref={el=>this.dom=el} style={{minHeight: 300}}>
          loading...
        </div>
      }
    }
  }
}

# 4 函数式组件需要注意到的问题

对于 classCompoennt 的组件，我们可以无感知的使用这个HOC直接包裹，但是对于 functionComponent 来说就不行了，因为 functionComponent 是没有实例的。 不过还好react hooks的出现解决了这个问题。

想要使用这个HOC包裹 functionComponent 的话，需要使用 React.forwardRef 和 React.useImperativeHandle 两个api。直接上代码：

import React, { Ref, useState } from 'react'
import { Card } from 'antd'
import { HomeData } from '../../models/connect'
import lazyLoadCom from '../../hoc/LazyLoadCom'

interface DataItemProps {
  data: HomeData;
  getData: ()=>void;
}

const DataItem = (props, ref)=> {
  const [count, setCount] = useState(0)
  const refDom = React.useRef(null)
  
  // 关键 设置ref
  React.useImperativeHandle(ref, ()=>refDom.current)

  return (
    <Card
      ref={refDom}
      title={props.data.name}
      style={{margin: '24px', minHeight: 300}}
      extra={<a onClick={props.getData}>刷新数据</a>}
    >
      表格值为：{props.data.value}
        <div>
        <div>{count}</div>
          <button onClick={()=>setCount(count+1)}>add</button>
        </div>
    </Card>
  )
}

const DataItemWarp = React.forwardRef<any, DataItemProps>(DataItem)

export default lazyLoadCom(DataItemWarp)
