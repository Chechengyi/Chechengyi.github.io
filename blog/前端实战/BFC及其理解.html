# BFC及其理解

BFC(Block formatting context)直译为"块级格式化上下文"。它是一个独立的渲染区域，只有Block-level box参与， 它规定了内部的Block-level Box如何布局，并且与这个区域外部毫不相干。

说真的，看到这句对BFC概念的阐述，我真的不知道再说什么东西，甚至怀疑我到底有没有学过css，诚然我的css是很差的那一种，但是这样空泛的概念确实对于我对BFC的理解只起到了又蒙上一层猪油的作用。接下来记录一下利用BFC的特性能做什么操作。

# 1 子元素设置margin，父元素也受影响

产生这个的原因是 margin重叠

所有毗邻的两个或者更多盒子元素的margin将会合并为一个margin共同享之，毗邻的定义为：同级或者嵌套的盒子元素，并且它们之间没有非空内容、padding、border隔离(经实践，如果是同级元素，有padding、border也不行，如果是同级元素，他们之间的margin取较大的一个)。

定义已经说的很清楚， 嵌套属于毗邻，以在样式表中优先级更高子元素的margin会覆盖外层父元素定义的margin。

父元素的第一个子元素的上边距margin-top如果碰不到有效的border或者padding。就会不断一层一层的找自己 “领导”(父元素，祖先元素)的麻烦。只要给领导设置个有效的 border或者padding就可以有效的管制这个目无领导的margin防止它越级，假传圣旨，把自己的margin当领导的margin执行。

所以遇到这个问题的解决办法为：

  1. 是父元素产生一个BFC，设置父元素 overflow:hidden，设置浮动或者其他能产生一个BFC的操作。因为产生了 BFC 后，BFC内的元素不应对外部的元素布局在产生影响。
  2. 给父元素设置 border 或者 padding。

# 2 如何形成一个BFC，以及BFC的布局规则

在看利用BFC能做什么操作之前，先看一下如何创建一个BFC。以及看BFC的特性

  • float的值不是none。
  • position的值不是static或者relative。
  • display的值是inline-block、table-cell、flex、table-caption或者inline-flex
  • overflow的值不是visible

BFC布局规则

  • 内部的Box会在垂直方向，一个接一个地放置。
  • BFC的区域不会与float box重叠。
  • BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
  • 计算BFC的高度时，浮动元素也参与计算。

# 2 根据BFC特性所作的实际操作

# 2.1 BFC的区域不会于 float box重叠

问题：要实现一个左边固定，右边自适应布局 此问题我们采用将左边的盒子 float:left 来做：

<!DOCTYPE html>
<html>
<head>
	<title>BFC盒子不会与float-box发生重叠</title>
	<style type="text/css">
		#box1 {
			width: 200px;
			height: 200px;
			background-color: blue;
			float: left;
		}
		#box2 {
			height: 500px;
			background: pink;
		}
	</style>
</head>
<body>
	<div>
		<div id="box1">浮动盒子</div>
		<div id="box2">
			非浮动盒子
		</div>
	</div>
</body>
</html>

写下代码用浏览器打开文件，会发现呈现出来的效果是： 可以看到，浮动盒子和正常的盒子重合了。此时利用BFC的特性，BFC的区域不会于 float box重叠。给 box2 加上一个 overflow:hidden属性。就可以看到两个区域不重叠了。

# 2.2 使用BFC解决元素浮动后，父元素高度坍塌问题

如果一个元素的子元素全部都设置了浮动的话，该父元素的高度就会消失，在清除浮动的手段中可以设置父元素：overflow:hidden，此时也是让父盒子形成一个 BFC， 计算BFC的高度时，浮动元素也参与计算 就是根据这条特性清楚浮动。

# 2.3 box垂直方向的距离由margin决定，属于同一个BFC的两个相邻box的margin会发生重叠

代码：

<!DOCTYPE html>
<html>
<head>
	<title>margin重叠</title>
	<style type="text/css">
		#box1 {
			width: 200px;
			height: 200px;
			background: pink;
			margin-bottom: 50px;
		}
		#box2 {
			width: 200px;
			height: 200px;
			background: blue;
			margin-top: 100px;
			overflow: hidden;
		}
	</style>
</head>
<body>
	<div id="box1"></div>
	<div id="box2"></div>
</body>
</html>

接下来观察两个盒子之间的距离。

可以看到两个盒子的 margin 重叠了，值为较大的100.

属于同一个 BFC 内的盒子之间的边距由距离较大的那个决定

既然是由于同属一个 BFC 内有这个特性，那现在尝试让这两个盒子不属于同一个 BFC 内。接下来给 box2 加上一个父盒子，并且设置为 BFC.

<!DOCTYPE html>
<html>
<head>
	<title>margin重叠</title>
	<style type="text/css">
		#box1 {
			width: 200px;
			height: 200px;
			background: pink;
			margin-bottom: 50px;
		}
		#box2 {
			width: 200px;
			height: 200px;
			background: blue;
			margin-top: 100px;
			overflow: hidden;
		}
	</style>
</head>
<body>
	<div id="box1"></div>
	<div style="overflow: hidden;">
		<div id="box2"></div>
	</div>
</body>
</html>

# 3综合看待，margin重叠加子元素影响父元素margin

<!DOCTYPE html>
<html>
<head>
	<title>margin重叠加子元素设置margin父元素受影响</title>
	<style type="text/css">
		#box1 {
			width: 200px;
			height: 200px;
			background: blue;
			margin-bottom: 100px;
		}
		#box2 {
			background: red;
			width: 300px;
			height: 300px;
		}
		#box3 {
			background: pink;
			width: 200px;
			height: 200px;
			margin-top: 50px;
		}
	</style>
</head>
<body>
	<div id="box1"></div>
	<div id="box2">
		<div id="box3"></div>
	</div>
</body>
</html>

这段代码最终呈现出来的结果是：box1 与 box2 之间的间距为 100px，box2 与 box3 之间没有间距。用刚才整理的知识解释一下就是：

  1. 首先，box2 与 box3 之间的关系为嵌套毗邻，由于现在 box2 没有border和padding，所以姑且可以认为 box2 受到了 box3 的影响，此时的 margin-top 为 50px
  2. box1 和 box2 属于同一个 BFC内（html），所以它们之间的间距会发生重叠，取较大的那一个：100px。

此时，如果将 box2 设置 overflow:hidden 盒子 box1 与 盒子 box2 之间的间距不会改变，盒子 box2 与 box3 之间产生 50px 艰巨，因为此时 box2 已经形成 BFC 了，box3 的间距不应该在影响 box2 对于外部元素的间距，这印证了 BFC 的一点，BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素，反之也如此

此时再来总结一下 BFC 的布局规则带给我们的一些实际操作： BFC布局规则

  • 内部的Box会在垂直方向，一个接一个地放置。
  • BFC的区域不会与float box重叠(两栏布局，左边固定、右边自适应等)。
  • BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此(解决子元素margin影响父元素margin)。
  • 计算BFC的高度时，浮动元素也参与计算(解决由于元素浮动带来的父元素高度坍塌问题)。
