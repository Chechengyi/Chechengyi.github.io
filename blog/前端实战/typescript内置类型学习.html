# typescript内置类型学习

现在 typescript 是越来越普及了。前几天面试的时候也遇到了问这方面的问题。

面试官：我看你项目经历中有说到使用了 typescript 你用到什么程度了？

我：就常用的把，在最初使用 typescript 的过程感觉克服的最大的困难就是泛型，但是理解了就还好。之前在这方面有一个卡的比较久的点就是比如此时使用高阶组件包裹A组件后，在其他的地方引用A组件就失去类型提示了，后面了解到泛型就知道应该将A组件的props类型作为泛型传进高阶组件，然后高阶组件返回的时候定义返回组件的props，还有一个问题就是高阶组件可能回给被包裹的组件提供一些方法或值，那么在定义组件A的props的时候也需要定义进去，这种情况下如果只是简单的传递props类型，那么在引入组件A的地方就会多出高阶组件提供的方法或者值的提示，如果不是非必须的定义的话，还会报错。所以这时候我写了一个自定义类型，就是有类型A和类型B，过滤掉A中 与B的交集。

面试官：好，那你写一下你当时是如何实现的把

。。。我特么当时居然脑海一片空白忘记了。。。 确实半年多没写过ts的我忘记这个当时是怎么写的了，只记得用了一个 typescript 的内置类型。真是太尴尬了。所以自己写一片文章记录以下 typescript 的一些内置类型以及一结合 react 的一些实践把。

# Partial<T>、Readonly<T>

type Partial<T> = {
    [P in keyof T]?: T[P];
};
type Readonly<T> = {
    readonly [P in keyof T]: T[P];
};

Partial: 接收一个类型，将这个类型所有属性都标记为可选属性。

Readonly: 接收一个类型，将这个类型的所有属性标记为只读属性

# Pick<T, K>

type Pick<T, K extends keyof T> = {
    [P in K]: T[P]
}

接收一个类型T 和一个属性K，从类型T中选出属性K列举的属性组成一个类型

如：

interface User {
  name?: string;
  age: number;
  pwd: string;
  date: string
}

type PickUser = Pick<User, 'name'|'pwd'>

/* 
type PickUser = {
    name?: string;
    pwd: string;
}
*/

# Record<K, T>

type Record<K extends keyof any, T> = {
    [P in K]: T
}

K为属性，T为类型，返回一个类型以K中列举的属性做key，T做value

interface User {
  id: string;
  pwd: string;
}

type UserList = Record<'key1'|'key2', User>
type UserList1 = Record<number, User>

const userList: UserList = {
  key1: {id: '1', pwd: '1'},
  key2: {id: '1', pwd: '2'}
}
const UserList1 = {
  1: {id: '1', pwd: '1'},
  2: {id: '1', pwd: '2'}
}

# Exclude<T, U>

type Exclude<T, U> = T extends U ? never : T;

接收两个联合属性 T，U， 从T中剔除与U的相同属性。

// 'a' | 'd'
type A = Exclude<'a'|'b'|'c'|'d' ,'b'|'c'|'e' >

# Omit<T, K>

根据上文 Exclude 可以得到 Omit 的实现：

type Omit<T, K extends keyof P> = Pick<T, Exclude<keyof T, K>>

使用：

interface A {
  a: string;
  b: string;
  d: string;
}

interface B {
  b: string;
  c: string;
}

type C = Omit<A, keyof B>

Omit 就是接收一个类型T，和属性集合K，删除类型T中的属性K，返回这个类型。

我也写过一个类似功能的自定义类型：

type Filter<A, B> = {
   [K in Exclude<keyof A, keyof B>]: A[K]
}

与Omit不同的是，这个接收的是第一个参数是类型，第二个参数也是类型。

type Filter<A, B> = {
  [K in Exclude<keyof A, keyof B>]: A[K]
}
// 这里的Filter 不适用于传入这样的联合属性
// type Ceshi1 = 'a' | 'b' | 'c'
// type Ceshi2 = 'c' | 'd'
type Ceshi1 = {
  a: string;
  b: string;
  c: string;
}
type Ceshi2 = {
  c: string;
  d: string;
}
type Ceshi3 = Filter<Ceshi1, Ceshi2>

其实比起 typescript 的 Omit 就是第二个参数少写了一个 keyof。

# ReturnType

如何使用 ReturnType 呢？

type A = ()=>string
type B = ReturnType<A>   // B string

ReturnType 的实现：

type ReturnType<T> = T extends (...args: any)=>infer P ? P : any

这个类型大概的意思就是，看看泛型 T 能不能赋值给 (...args: any)=>infer P ，如果可以的话推断出 P 类型，否则是 any。这里有一个关键字，infer，inder P 表示待推断的类型。

由此，其实我们自己也可以利用 infer 关键字来写出一些类型，比如，写一个获取函数的第一个入参的类型的：

type ParamsType<T> = T extends (param: infer P)=>any ? P : any

然后使用：

type P1 = (name: string) => void
type P2 = ParamsType<P1>  // P2 string
type P3 = ParamsType<number> // P3 any ，number不符合 (param: infer P)=>any
