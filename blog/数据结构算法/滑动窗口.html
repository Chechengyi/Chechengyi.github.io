# 滑动窗口的最大值

示例 1：

输入：nums = [1,3,-1,-3,5,3,6,7], k = 3
输出：[3,3,5,5,6,7]
解释：
滑动窗口的位置                最大值
---------------               -----
[1  3  -1] -3  5  3  6  7       3
 1 [3  -1  -3] 5  3  6  7       3
 1  3 [-1  -3  5] 3  6  7       5
 1  3  -1 [-3  5  3] 6  7       5
 1  3  -1  -3 [5  3  6] 7       6
 1  3  -1  -3  5 [3  6  7]      7

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/sliding-window-maximum
著作权归领扣网络所有。商业转载请联系官方授权，非商业转载请注明出处。

谜底就出在谜面上，这道题的题目就叫做滑动窗口的最大值。最终题目的答案就是在 k 大小的滑动窗口在从最左端移动到最右端的时候，滑动窗口中出现的最大值，最终返回一个一维数组。

掌握这道题，能让我们熟悉滑动窗口在向右移动的过程中的基本的操作，但是其实解决这道题的关键思想在于构建一个“单调队列”。

在 leetcode 的很多题解中都称为“单调栈”，但其实在写的过程中发现这个数据结构左能出右能进，我觉得称为队列更合理一点。这个队列它保存的就是数组的下标，而且它保证了一点，就是最左端的下标所对应的值是这个队列中最大的。这样做就使得，我们只要取最左端的值，就能得到当前滑动窗口的最大值。当然，在滑动窗口向右移动的过程中，我们还需要注意判断最左端的值是否已经不处于滑动窗口的视野当中，以及右侧进入的新的数入队列。接下来我们用这个思想模拟一下例子。

我们设“单调队列”为task。滑动窗口的左侧为left，右侧为right，初始的时候left=right=0。

首先，是要使滑动窗口的大小等于k（这里为3），1进入task，然后3进入task但是3比1大所以1 pop()，然后-1进入task（注意task里面保存的是下标）。此时已经得到了第一个滑动窗口中的最大值，那就是task[0]，下标为1，所以值是3.

从现在开始，滑动窗口开始往右移动了。此时right=3了，首先将进进入滑动窗口视野中的元素-3插入到队列当中。然后此时left=1，看看task最左侧的下标是否还处于滑动窗口的视野当中，如果不处于则要将其 shiht()。然后当前滑动窗口的最大值的下标还是 task[0]，重复这样的操作，直到滑动窗口移动到最右端，这样就能得到答案了。接下来上代码：

var maxSlidingWindow = function(nums, k) {
    let len = nums.length
    let queue = []

    for ( let i=0; i<k; i++ ) {
        while ( queue.length && nums[i] >= nums[queue[queue.length-1]] ) {
            queue.pop()
        }
        queue.push(i)
    }


    let ret = []
    ret[0] = nums[queue[0]]

    for ( let i=k; i<nums.length; i++ ) {
        // 将进入到滑动窗口的元素push到 队列当中
        while( queue.length && nums[i] >= nums[queue[queue.length-1]] ){
            queue.pop()
        }
        queue.push(i)

        // 如果当前队列中最左侧的下标已经不再滑动窗口内，删除
        while(queue[0] <=  (i-k)){
            queue.shift()
        }
        ret.push(nums[queue[0]])
    }
    return ret
};

# 可获得的最大点数

几张卡牌 排成一行，每张卡牌都有一个对应的点数。点数由整数数组 cardPoints 给出。

每次行动，你可以从行的开头或者末尾拿一张卡牌，最终你必须正好拿 k 张卡牌。

你的点数就是你拿到手中的所有卡牌的点数之和。

给你一个整数数组 cardPoints 和整数 k，请你返回可以获得的最大点数。

示例 1：

输入：cardPoints = [1,2,3,4,5,6,1], k = 3
输出：12
解释：第一次行动，不管拿哪张牌，你的点数总是 1 。但是，先拿最右边的卡牌将会最大化你的可获得点数。最优策略是拿右边的三张牌，最终点数为 1 + 6 + 5 = 12 。

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/maximum-points-you-can-obtain-from-cards
著作权归领扣网络所有。商业转载请联系官方授权，非商业转载请注明出处。

换一个角度去思考这个问题，每次拿牌只能从行的开头或者末尾去拿，站在数组的角度去考虑，那么最终剩下的牌一定是连续的子数组，且这个数组的长度为 cardPoints.length - k，而总的点数是不会改变的，所以可以将问题转化为：在 cardPoints 中找出一个长度为 cardPoints.length - k 的数组其和最小。最终答案就是数组总和减去最小滑动窗口总和。接下来直接贴出代码：

var maxScore = function(cardPoints, k) {
  let totalSum = cardPoints.reduce((a,b)=>a+b)
  let windowSize = cardPoints.length - k
  if ( !windowSize ) return totalSum
  let windowSum = 0
  let minWindowSum = 0  
  let left = 0, right = 0
  while(right < windowSize) {
    windowSum += cardPoints[right]
    right++
  }
  minWindowSum = windowSum
  while( right < cardPoints.length ){
  	windowSum = windowSum - cardPoints[left] + cardPoints[right]
  	minWindowSum = Math.min(windowSum, minWindowSum)
  	right++
  	left++
  }
  return totalSum - minWindowSum
};

# 替换后的最长重复字符串

给你一个仅由大写英文字母组成的字符串，你可以将任意位置上的字符替换成另外的字符，总共可最多替换 k 次。在执行上述操作后，找到包含重复字母的最长子串的长度。

注意：字符串长度 和 k 不会超过 104。

示例 1：

输入：s = "ABAB", k = 2
输出：4
解释：用两个'A'替换为两个'B',反之亦然。

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/longest-repeating-character-replacement
著作权归领扣网络所有。商业转载请联系官方授权，非商业转载请注明出处。

这个其实也是一个求字符串字串的问题。让我们来思考一下，假如现在有一个字符串，我能做k次替换操作，现在想要让这个字符串中的 重复字母尽可能的多，那么我的做法应该是找到这个字符串中重复字母最多的那一个然后把其他字符替换成这个字符。 所以，对应到这个题当中，毫无疑问我们要保留的字符就是当前出现次数最多的字符，而题目又要求了最终要的结果是连续的字符串，所以，这就限制了字符串的长度（也就是滑动窗口的大小）。接下来直接上代码：

var characterReplacement = function(s, k) {
  // 用来记录滑动窗口内字母出现的次数
  let map = new Array(26).fill(0)
  let left = 0, right = 0, max = 0
  let ret = 0  // 用来记录最优的解 
  while( right < s.length ){
    // 一个字母进入窗口，在map中将次数加一，并且更新最大字母重复次数
    let char = s[right]
    let index = char.charCodeAt() - 'A'.charCodeAt()
    map[index]++
    max = Math.max(max, map[index])
    // 判断当前窗口的字符串是否符合规则，
    // 如果当前窗口长度减去最大字母出现次数 的值 大于最大替换次数 K
    // 则不符合规则 所以整个窗口往左移动，left++ 且要将 map中记录的值减去
    if ( right - left + 1 - max > k ) {
      map[s[left].charCodeAt() - 'A'.charCodeAt()]--;  
      left++
    }
    right++
    ret = Math.max(ret, right-left)
  }
  return ret
  // return right - left
};

其实这段代码还有一个可以优化的点，那就是不需要使用 ret 来记录最优解。因为每一次循环right都在向右移动，不符合条件的时候left才会向左移动，这说明 right-left 的值一定是递增的。所以最后 right-left 一定是最优解。
