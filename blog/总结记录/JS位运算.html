# JS位运算

最近 leetcode 的每日一题，好多都是跟位运算有关的，今天来总结一下 JS 中位运算的一些概念，以及在实际使用中的一些用处和技巧。

以 JS 为例子，JS中对于数字只有一个 number 类型，在底层将数字存储为64位浮点数 而在执行位运算之前，JS 会将数字转换为 32 位有符号整数，执行按位操作后，结果将转换回 64 位浮点数。

不同于 Java 这样的语言，JS 数字不区分整型和浮点型，数字均以 64 位浮点格式表示。而 Java 中的 int 类型则是以 32 位的有符号数字表示。但是 JS 在执行位运算的过程中，会将数字转成 32 位有符号整数。

既然我们现在关心的是位运算，那就看看一个整数使用 32 位有符号整数是怎么去存储的。有符号整数使用 31 位表示整数的数值，第 32 位表示整数的符号，0 表示正数，1 表示负数。如，正数18的表示方法如下：

# 按位非 (NOT)

操作符由一个波浪线 ~ 表示对位求反，也就是原本是 0 变成 1，原本是 1 变成 0。

  • 对一个整数 num 按位取反，结果等于 它的相反数减1，即 ~num = -num - 1。
  • 对一个整数两次按位非，可以得到它本身；对一个小数两次按位非，可以得到取整效果。

# 按位与 (AND)

操作符为 &，只有两个数值对应位都为 1 的情况下才能得到 1。

位数 第4位 第3位 第2位 第1位
数字a 0 0 1 1
数字b 1 1 0 1
结果(&) 0 0 0 1
  • 可以用 & 来验证数字的奇偶性，将数 num 和 1 &，若结果为 1，则 num 为奇数，若结果为 0，则数为偶数。

# 按位或 (OR)

操作符号为：|，两个数字对应位 的值都为 0 才能得到 0，否则 为 1。

位数 第4位 第3位 第2位 第1位
数字a 0 1 0 1
数字b 0 1 1 0
结果(|) 0 1 1 1

# 按位异或

操作符号为：^，两个数字对应位置 的值不同时为1，否则为0。

位数 第4位 第3位 第2位 第1位
数字a 0 1 1 0
数字b 0 0 1 1
结果(^) 0 1 0 1
  • 整数与它本身按位异或的结果都是 0。
  • 整数与 0 按异或得到其本身。

# 左移

操作符：<< 。会将所有的位数向左移动指定的位数，右边的空位用0填充。左移不影响数的取整效果。

# 有符号右移

操作符：>>，会将所有数值的位数向右移动指定的位数，同时保留该数的符号。

# 无符号右移

操作符：>>>，会将数值的所有位向右移动指定的位数。无符号右移运算用 0 填充所有空位。对于正数，这与有符号右移运算的操作一样；而负数则被作为正数来处理。

# 2.1 求汉明距离

两个整数之间的汉明距离指的是这两个数字对应二进制位不同的位置的数目。

给出两个整数 x 和 y，计算它们之间的汉明距离。

示例:

输入: x = 1, y = 4

输出: 2

解释:
1   (0 0 0 1)
4   (0 1 0 0)
       ↑   ↑

上面的箭头指出了对应二进制位不同的位置。

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/hamming-distance

对两个数字进行异或，则相同位的值都变成了 0，而不同位的值都变成了 1，然后就是如果计算出该二进制数中有多少个1，JS 中并没有提供可以直接获取二进制中有多少个1的 API，不过可以通过 &1 的操作，验证第一位是否为1，然后将数字右移动一位在继续验证。

var hammingDistance = function(x, y) {
  // 先使用异或
  let s = x ^ y
  let ret = 0

  while (s>0){
    // 在使用 & 检验最后一位是否为1，1 为ob01，如果最后一位为1 则结果为1，反之
    ret += s & 1
    s >>= 1
  }

  return ret
};

# 2.2 React 中对于位运算的应用

如果有了解过 React 的源码，应该对 effect 这个概念比较熟悉。简单来说，React 在其 Fiber 对象上使用了一个 effectTag 属性来记录该节点将会有哪些操作。比如 Deletion 是一个删除操作，Snapshot 则是 ClassComponent 中的 getDerivedStateFromProps 。通过使用位运算，记录当前节点具有哪些操作，借助位运算，能够高效的添加、删除、验证。

比如现在如果要为 一个节点增加 Snapshot 操作，只需要按位或：effectTag |= Snapshot ，如果之后还需要添加一个 Ref 操作，则 effctTag |= Ref。 在验证节点是否具有相应的操作时，使用按位与操作： if ( effectTag & Snapshot ) ，如果 effectTag 设置过 Snapshot 的，则 effectTag 的第 9 位一定是 1，则 Fiber.effectTag & Snapshot 的值就会大于 0。

如果现在想要删除 其中一个操作应该怎么做呢？比如现在要删除掉 effectTag 中的 Snapshot ，则：effectTag &= effectTag ^ Snapshot。

接下来来验证一下这样的位运算是否真的正确，这里虽引出了 React 中的应用，但是位数太多了，不方便我们举证。接下来我们使用赋予 “增删查改” 四个操作来测试一下。首先定义好每个操作对应的数值：
