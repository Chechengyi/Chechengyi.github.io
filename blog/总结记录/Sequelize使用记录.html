# Sequelize 使用记录

sequelize 是一款强大的 orm 工具。写下本篇文章记录一下 sequelize 使用过程。

此处先贴出一条终极建议：

在使用 Sequelize 后，数据库字段的命名请尽可能地使用下划线命名的方式。我发现 Sequelize 在会将驼峰的命名自动的转换为下划线的命名方式，这个坑让我在多对多定义外键的时候头皮都差点抓掉了，如果想要接口输出到前端美观是驼峰式命名，可以在 model 定义的时候添加 field 属性

# 1、Sequelize 迁移工具 Migrations

其实之前也略微接触过 Sequelize，那时候生成数据库表是直接写model，然后运行 .sync 方法。这样的做法在个人小项目中使用也显得很方面，但是如果是多人协作的项目，或者是一个需要长期去维护的项目，这样做劣势比较明显，很简单的一点，随着项目的迭代，有很多时候需要改数据库表，比如增加字段什么的。如果使用这样的方法，使得开发人员增加的行为根本无法追踪。我认为，使用 Migrations 的意义在于，规范开发人员对于数据库修改操作。

# 1.1 安装 sequelize-cli 和初始化

npm install --save sequelize-cli

然后在项目中就可以以：

npx sequelizt xxx

在实际的项目中，我们希望所有与数据库 Migrations 相关的内容都放到 database 目录下，所以在项目根目录下新建一个 .sequelizerc 配置文件：

'use strict';

const path = require('path');

module.exports = {
  config: path.join(__dirname, 'database/config.json'),
  'migrations-path': path.join(__dirname, 'database/migrations'), // 迁移脚本存放文件
  'seeders-path': path.join(__dirname, 'database/seeders'), // 种子脚本文件
  'models-path': path.join(__dirname, 'app/model'), // 模型文件
};

然后初始化 Migrations 和目录：

npx sequelize init:config
npx sequelize init:migrations

这样会在项目的根目录下生成一个 database 文件夹，里面有一个 config.json 文件：

{
  "development": { // 开发环境数据库
    "username": "root",
    "password": "",
    "database": "test", // 数据库名称
    "host": "localhost",
    "dialect": "mysql"
  },
  "test": {    // 测试数据库
    "username": "root",
    "password": "",
    "database": "test1",
    "host": "localhost",
    "dialect": "mysql"
  },
  "production": { // 生产环境数据库
    "username": "root",
    "password": null,
    "database": "database_production",
    "host": "127.0.0.1",
    "dialect": "mysql"
  }
}

还会生成一个migrations 文件夹，这个文件夹下就存放着我们将要进行的迁移脚本文件。

# 2.1 编写model

首先第一步是要确定模型，我们在 .sequelizerc 配置文件中所声明的 models-path 下编写模型。

User model：


export default (app: Egg.Application) => {
  const { STRING, INTEGER, DATE } = app.Sequelize;

  const User = app.model.define('user', {
    id: { type: INTEGER, primaryKey: true, autoIncrement: true },
    name: STRING(30),
    age: INTEGER,
    created_at: DATE,
    updated_at: DATE
  });

  User.associate = function(models){
    User.belongsToMany(models.Role, {
      as: 'roles',
      through: models.UserRole,
      foreignKey: 'user_id'
    })
  }

  return User
}

Role model:

export default (app: Egg.Application)=> {
  const { STRING, INTEGER, DATE } = app.Sequelize;
  const Role = app.model.define('role', {
    id: { type: INTEGER, primaryKey: true, autoIncrement: true },
    name: STRING(30),
    created_at: DATE,
    updated_at: DATE
  })

  Role.associate = function(models){
    Role.belongsToMany(models.User, {
      as: 'users',
      through: models.UserRole,
      foreignKey: 'role_id'
    })

}

UserRole model:

  })
  return UserRole

# 2.2 编写迁移文件

接下来编写迁移文件，执行：

npx sequelize-cli migration:create --name User

它会在 migrations 目录下自动创建一个时间-表名.js 的脚本文件。

user 的迁移脚本：

'use strict';

module.exports = {
  up: async (queryInterface, Sequelize) => {
    /**
     * Add altering commands here.
     *
     * Example:
     * await queryInterface.createTable('users', { id: Sequelize.INTEGER });
     */
    const { INTEGER, DATE, STRING } = Sequelize
    return queryInterface.createTable('users', {
      id: { type: INTEGER, primaryKey: true, autoIncrement: true },
      name: STRING(30),
      age: INTEGER,
      created_at: DATE,
      updated_at: DATE,
    })
  },

  down: async (queryInterface, Sequelize) => {
    /**
     * Add reverting commands here.
     *
     * Example:
     * await queryInterface.dropTable('users');
     */
    await queryInterface.dropTable('users');
  }
};

role的迁移脚本：

'use strict';

module.exports = {
  up: async (queryInterface, Sequelize) => {
    /**
     * Add altering commands here.
     *
     * Example:
     * await queryInterface.createTable('users', { id: Sequelize.INTEGER });
     */
    const { INTEGER, DATE, STRING } = Sequelize
    return queryInterface.createTable('roles', {
      id: { type: INTEGER, primaryKey: true, autoIncrement: true },
      name: STRING(30),
      created_at: DATE,
      updated_at: DATE
    })
  },

  down: async (queryInterface, Sequelize) => {
    /**
     * Add reverting commands here.
     *
     * Example:
     * await queryInterface.dropTable('users');
     */
    await queryInterface.dropTable('roles');
  }
};

user_role的迁移脚本：

'use strict';

module.exports = {
  up: async (queryInterface, Sequelize) => {
    /**
     * Add altering commands here.
     *
     * Example:
     * await queryInterface.createTable('users', { id: Sequelize.INTEGER });
     */
    const { INTEGER, STRING } = Sequelize
    return queryInterface.createTable('user_role', {
      id: { type: INTEGER, primaryKey: true, autoIncrement: true },
      name: STRING(30),
      user_id: {
        type: INTEGER,
        allowNull: false,
        references: {
          model: 'users',
          key: 'id'
        }
      },
      role_id: {
        type: INTEGER,
        allowNull: false,
        references: {
          model: 'roles',
          key: 'id'
        }
      }
    })
  },

  down: async (queryInterface, Sequelize) => {
    /**
     * Add reverting commands here.
     *
     * Example:
     * await queryInterface.dropTable('users');
     */
    await queryInterface.dropTable('user_role');
  }
};

将脚本文件写好后，执行：

npx sequelize db:migrate

接下来去数据库中查看，就可以看到生成了三张表。

# 2.3 表的更新

在实际的开发过程中，很多时候需要给一张表添加字段，这时候需要重新编写更新表的迁移脚本。比如，现在我想要在 User 表中添加一个性别字段，这个字段为sex。首先，使用命令添加一个新的迁移脚本：

npx sequelize-cli migration:create --name User-AddSex

然后编写脚本文件：

'use strict';

module.exports = {
  up: async (queryInterface, Sequelize) => {
    /**
     * Add altering commands here.
     *
     * Example:
     * await queryInterface.createTable('users', { id: Sequelize.INTEGER });
     */
    return queryInterface.addColumn('users', 'sex', {
      type: Sequelize.INTEGER
    })
  },

  down: async (queryInterface) => {
    /**
     * Add reverting commands here.
     *
     * Example:
     * await queryInterface.dropTable('users');
     */
    return queryInterface.removeColumn('users', 'sex')
  }
};

然后在执行: npx sequelize db:migrate。就完成了更新表的操作，当然，别忘记了记得在 model 处也映射新添加的字段。

需要注意的是，更新表的时候，一定要写新的迁移脚本文件，npx sequelize db:migrate 命令只会执行没有运行过的脚本文件。这样做可以清晰的看到表的操作过程，将每一次的更新隔离开，也有助于回滚操作。通过：db:migrate:undo 命令则可以执行回退操作，就是执行迁移文件里面的 down 方法。

# 2.3 添加种子

这个操作其实就是往数据库里面添加一些数据。

npx sequelize-cli seed:generate --name demo-user

执行这个命令会在 migrations 文件夹下生成一个 seeders 文件夹，里面会生成 xxxxx-demo-user.js 文件，很显然，这是要向 User 表里面插入的数据，

然后执行命令将种子插入到数据库中：npx sequelize-cli db:seed:all

# 2.4 查询

假如，现在我想要在查询用户的时候，顺便也查出当前用户拥有哪些角色应该怎么做呢？以在 egg.js 中的 sevice 层操作为例子：

const users = await this.app.model.User.findAll({
      include: [{
        model: this.app.model.Role,
        as: 'roles'
      }]
    })
    return users;

在查询的时候使用添加一个 include 属性，注意，这里的 as: roles 必须与在定义外建关系的时候设置的 as 命名相同：

User.belongsToMany(app.model.Role, {
      as: 'roles',
      through: app.model.UserRole,
      foreignKey: 'user_id'
    })
